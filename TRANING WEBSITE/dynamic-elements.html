JavaScript - Creating Elements Dynamically

New elements can be dynamically created in JavaScript to build interactive and highly responsive web pages. By using the DOM API, you can generate HTML tags, add text, apply attributes, and place them exactly where you need them without reloading the page.

1. The createElement() Method

The document.createElement() method creates an HTML element specified by its tag name. Once created, the element exists in memory but is not yet visible on the page until you append it.

<!DOCTYPE html> <html> <body> <h3>Creating a New Paragraph</h3> <div id="container1" style="border: 2px dashed #04AA6D; padding: 15px;"></div> <br> <button onclick="addParagraph()">Generate Element</button> <script> function addParagraph() { // 1. Create the element let newPara = document.createElement("p"); // 2. Add content newPara.innerText = "This paragraph was created dynamically!"; // 3. Append to the DOM document.getElementById("container1").appendChild(newPara); } </script> </body> </html>

2. Using createTextNode()

Instead of setting innerText, you can create a standalone text node using document.createTextNode() and append it to your newly created element. This is the safest way to add text without parsing it as HTML.

<!DOCTYPE html> <html> <body> <h3>Dynamic List Items</h3> <ol id="parentList"> <li>Existing Item 1</li> </ol> <button onclick="addItem()">Add List Item</button> <script> function addItem() { let listItem = document.createElement("li"); let textNode = document.createTextNode("Newly generated list item"); listItem.appendChild(textNode); document.getElementById("parentList").appendChild(listItem); } </script> </body> </html>

3. Setting Attributes with setAttribute()

After creating an element, you often need to define its attributes (like classes, IDs, or source links). The setAttribute() method allows you to apply these dynamically.

<!DOCTYPE html> <html> <head> <style> .dynamic-box { background-color: #04AA6D; color: white; padding: 10px; margin-top: 10px; } </style> </head> <body> <h3>Adding Attributes</h3> <div id="container3"></div> <br> <button onclick="createStyledBox()">Create Styled Box</button> <script> function createStyledBox() { let box = document.createElement("div"); box.innerText = "I have a dynamic class!"; // Set the class attribute box.setAttribute("class", "dynamic-box"); document.getElementById("container3").appendChild(box); } </script> </body> </html>

4. Inserting Elements (insertBefore)

The appendChild() method always adds the new element at the very end. If you want to place it somewhere specific, you use insertBefore(newNode, existingNode).

<!DOCTYPE html> <html> <body> <h3>Inserting at Specific Positions</h3> <ul id="myList"> <li id="secondItem">Item B</li> <li>Item C</li> </ul> <button onclick="insertAtTop()">Insert Item A</button> <script> function insertAtTop() { let newItem = document.createElement("li"); newItem.innerText = "Item A"; newItem.style.color = "red"; let parent = document.getElementById("myList"); let referenceNode = document.getElementById("secondItem"); // Insert newItem BEFORE referenceNode parent.insertBefore(newItem, referenceNode); } </script> </body> </html>

5. Removing Elements

To dynamically remove an element, you can use the remove() method on the element itself, or use removeChild() from the parent node.

<!DOCTYPE html> <html> <body> <h3>Removing Elements</h3> <div id="deleteMe" style="background: #333; color: white; padding: 15px;"> I am a temporary message block! </div> <br> <button onclick="deleteElement()">Remove Block</button> <script> function deleteElement() { let elementToRemove = document.getElementById("deleteMe"); if (elementToRemove) { elementToRemove.remove(); // Modern way to remove } else { alert("Element already deleted!"); } } </script> </body> </html>
GeeksforGeeks Pro Tip: Generating UI elements dynamically using createElement() rather than injecting massive innerHTML strings helps protect your applications against Cross-Site Scripting (XSS) vulnerabilities and makes event listener attachments significantly more manageable.
