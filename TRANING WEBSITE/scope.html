JavaScript Scope

Scope in JavaScript determines the accessibility or visibility of variables and functions in different parts of your code.

Scope Types Overview

Scope Type Description
Global Scope Accessible from anywhere in the script.
Function Scope Local to the function; inaccessible from outside.
Block Scope Variables (let/const) local to curly braces {}.

1. Global Scope

<script> let globalVar = "I am Global"; function testScope() { document.write(globalVar + " (accessed inside function)"); } testScope(); </script>

2. Function (Local) Scope

<script> function myFunction() { let localVar = "I am Local"; document.write(localVar); } myFunction(); </script>

3. Block Scope (let & const)

<script> { let blockVar = "Visible only inside this block"; document.write(blockVar); } </script>

4. The Scope Chain

JavaScript looks for variables in the current scope. If it doesn't find them, it moves to the outer scope until it reaches the global scope.

<script> let name = "Outside"; function outer() { function inner() { document.write("Inner function accessing: " + name); } inner(); } outer(); </script>
Pro Tip: Variables declared with var do NOT have block scope. Always use let and const to prevent accidental data leaks!
