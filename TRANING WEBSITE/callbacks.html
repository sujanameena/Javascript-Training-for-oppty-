JavaScript Callbacks

A callback is a function passed as an argument to another function. This technique allows a function to call another function and is essential for managing program flow.

1. Basic Callback Execution

In JavaScript, functions are "First-Class Citizens." They can be passed around like any other variable.

<!DOCTYPE html> <html> <body> <h2>JavaScript Callbacks</h2> <p id="demo"></p> <script> function displayer(some) { document.getElementById("demo").innerHTML = some; } function myCalculator(num1, num2, myCallback) { let sum = num1 + num2; myCallback(sum); } // Pass displayer as a callback myCalculator(5, 5, displayer); </script> </body> </html>

2. Callbacks with Array Methods

Many built-in JavaScript methods, like filter(), use callbacks to determine how to process data.

<!DOCTYPE html> <html> <body> <h2>Array Callback</h2> <p id="demo"></p> <script> const numbers = [4, 9, 16, 25]; function isOver10(num) { return num > 10; } const over10 = numbers.filter(isOver10); document.getElementById("demo").innerHTML = "Numbers > 10: " + over10; </script> </body> </html>

3. Asynchronous Callbacks

Callbacks are most powerful when used for tasks that take time, such as fetching data or waiting for a timer.

<!DOCTYPE html> <html> <body> <h2>Async Callback</h2> <p id="demo">Waiting...</p> <script> function myMessage() { document.getElementById("demo").innerHTML = "Callback executed after 3 seconds!"; } setTimeout(myMessage, 3000); </script> </body> </html>
Oppty Pro Tip: When passing a function as a callback, pass the name without parentheses. Using parentheses executes the function immediately instead of waiting.
