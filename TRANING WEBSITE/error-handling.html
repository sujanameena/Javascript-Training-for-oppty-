Handling Network Errors

In web development, things will go wrong. A user's internet might drop, a server might crash, or a URL might change. Error Handling is the art of catching these issues so your app doesn't break.

The Fetch Trap: A fetch() promise does not reject on HTTP errors (like 404 or 500). It only rejects if the network request fails entirely (e.g., DNS error or being offline).

1. The Professional Fetch Pattern

To handle errors correctly, you must check two things: if the promise rejected and if the response.ok property is false.

Type of Error Status How to Catch it
HTTP Error 404, 500, 403 Check if (!response.ok)
Network Error Offline, DNS Failure Use .catch() or try...catch
Data Error Invalid JSON Catch during response.json()

2. Basic Error Handling (.then / .catch)

This example tries to fetch a non-existent page to trigger a 404 error manually.

<!DOCTYPE html> <html> <body> <p id="basic-res">Waiting for request...</p> <script> fetch('https://jsonplaceholder.typicode.com/invalid-page-name') .then(response => { if (!response.ok) { // Manually throw error for HTTP 404 throw new Error("Page Not Found (404)"); } return response.json(); }) .then(data => { document.getElementById("basic-res").innerText = "Success!"; }) .catch(error => { document.getElementById("basic-res").innerHTML = "<b style='color:red'>Caught:</b> " + error.message; }); </script> </body> </html>

3. Modern Error Handling (Async / Await)

Using try...catch blocks makes your code much cleaner. This example simulates a user lookup.

<!DOCTYPE html> <html> <body> <button onclick="loadUser()">Fetch User #999</button> <p id="status"></p> <script> async function loadUser() { const statusEl = document.getElementById("status"); statusEl.innerText = "Loading..."; statusEl.style.color = "black"; try { const response = await fetch('https://jsonplaceholder.typicode.com/users/999'); if (!response.ok) { // This jumps straight to the 'catch' block throw new Error(`Server returned error code: ${response.status}`); } const data = await response.json(); statusEl.innerText = "Success: " + data.name; } catch (err) { statusEl.innerText = "Error: " + err.message; statusEl.style.color = "red"; } } </script> </body> </html>

4. Handling Data (JSON) Parsing Errors

Sometimes the URL is correct, but the data returned is not valid JSON (e.g., the server sends an HTML error page instead).

<!DOCTYPE html> <html> <body> <p id="json-msg">Click to test invalid data handling.</p> <button onclick="testJSON()">Parse Broken Data</button> <script> async function testJSON() { try { // We will mimic a response that isn't JSON const fakeResponse = new Response("<html>Not JSON</html>"); const data = await fakeResponse.json(); } catch (err) { document.getElementById("json-msg").innerHTML = "<span style='color:orange'>Warning:</span> Could not read data as JSON!"; } } </script> </body> </html>

5. Handling Timeouts

A request might take too long to complete. You can use AbortController to stop a request after a certain time.

<!DOCTYPE html> <html> <body> <button onclick="fetchWithTimeout()">Fetch with 10ms Timeout</button> <p id="tout-res"></p> <script> async function fetchWithTimeout() { const controller = new AbortController(); const id = setTimeout(() => controller.abort(), 10); // Abort after 10ms try { await fetch('https://jsonplaceholder.typicode.com/posts', { signal: controller.signal }); document.getElementById("tout-res").innerText = "Success!"; } catch (err) { document.getElementById("tout-res").innerText = "Request Timed Out!"; document.getElementById("tout-res").style.color = "red"; } } </script> </body> </html>
Oppty Academy Tip: In a real project, never just console.log an error. Always show a friendly message to the user (like "Please check your connection") so they aren't staring at a blank screen!
