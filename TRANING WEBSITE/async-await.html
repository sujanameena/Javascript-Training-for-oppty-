JavaScript Async / Await

The async and await keywords make asynchronous code look and behave a bit more like synchronous code. This makes it much easier to read and maintain.

1. The async Keyword

The async keyword is placed before a function to make it return a Promise. If the function returns a value, JavaScript automatically wraps that value in a resolved Promise.

<!DOCTYPE html> <html> <body> <h2>JavaScript async</h2> <p id="demo"></p> <script> async function myFunction() { return "Hello Students!"; } myFunction().then( function(value) { document.getElementById("demo").innerHTML = value; } ); </script> </body> </html>

2. The await Keyword

The await keyword can only be used inside an async function. It makes the function pause and wait for a Promise to resolve before moving to the next line.

<!DOCTYPE html> <html> <body> <h2>JavaScript await</h2> <p id="demo">Waiting for message...</p> <script> async function displayMessage() { let myPromise = new Promise(function(resolve) { setTimeout(function() { resolve("Welcome to Oppty Academy!"); }, 3000); }); document.getElementById("demo").innerHTML = await myPromise; } displayMessage(); </script> </body> </html>

3. Error Handling (Try/Catch)

Instead of using .catch(), you can use a try...catch block to handle errors in an async function.

<script> async function fetchData() { try { let response = await fetch('https://api.example.com/data'); let data = await response.json(); console.log(data); } catch (error) { console.error("Oops! Something went wrong: " + error); } } fetchData(); </script>
Oppty Pro Tip: Using async/await eliminates "Callback Hell" and makes your code much cleaner than using multiple .then() chains.
