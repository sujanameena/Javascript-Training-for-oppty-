JavaScript Function Types

In JavaScript, there are several ways to define functions. Choosing the right one depends on whether you need hoisting, a specific this context, or a concise syntax.

1. Function Declaration

This is the most traditional way to define a function. It uses the function keyword followed by a unique name.

Key Feature: Hoisting. Function declarations are hoisted to the top of their scope, meaning you can call the function even before you write it in your code.

function greet(name) { return "Hello " + name; } console.log(greet("Oppty Student"));

2. Function Expression

In a function expression, a function is created and assigned to a variable. The function itself can be anonymous (have no name).

Key Feature: Unlike declarations, expressions are not hoisted. You cannot use the function before the line where it is defined.

const add = function(a, b) { return a + b; }; console.log(add(5, 10));

3. Arrow Functions (ES6)

Introduced in ES6, arrow functions provide a shorter, more concise syntax for writing function expressions.

Key Feature: They do not have their own this binding, making them perfect for callbacks. If the function has only one statement that returns a value, you can omit the return keyword and curly braces.

const multiply = (a, b) => a * b; console.log(multiply(4, 5));

4. Anonymous Functions

As the name suggests, these are functions without a name. They are usually used as arguments for other functions (callbacks).

Usage: They are frequently seen in event listeners or array methods like map, filter, and forEach.

setTimeout(function() { console.log("This runs after 2 seconds"); }, 2000);

5. Immediately Invoked Function Expressions (IIFE)

An IIFE is a function that runs as soon as it is defined.

Purpose: It is primarily used to create a private scope and avoid polluting the global namespace with variables.

(function() { let message = "Private data"; console.log(message); })();

Comparison of Function Types

Type Hoisted? Short Syntax? Best Use Case
DeclarationYesNoGeneral purpose, reusable logic.
ExpressionNoNoWhen you want to limit scope access.
ArrowNoYesShort callbacks and this context.
AnonymousNoN/ATemporary tasks / Callbacks.
IIFEN/ANoPrivacy and initialization.
Oppty Academy Pro Tip: Modern JavaScript development heavily favors Arrow Functions for their readability, but Function Declarations are still essential when you need hoisting!
