HTTP Methods & Status Codes

Every time a browser talks to a server, it uses an HTTP Method to describe the action and receives an HTTP Status Code to describe the result. Understanding these is essential for debugging and building APIs.

1. Understanding HTTP Methods

HTTP methods (also called verbs) define the desired action to be performed on a specific resource. They tell the server whether you want to read, create, update, or delete data.

Method Description Idempotent?
GET Requests data from a specified resource. Should only retrieve data. Yes
POST Sends data to a server to create a new resource. No
PUT Sends data to a server to update or replace an existing resource. Yes
PATCH Applies partial modifications to a resource. No
DELETE Deletes the specified resource. Yes

2. HTTP Status Codes Breakdown

Status codes are 3-digit numbers returned by the server. They are grouped into five distinct classes to help developers identify the nature of the response at a glance:

Range Category Common Examples
1xx Informational 100 (Continue), 101 (Switching Protocols)
2xx Success 200 (OK), 201 (Created)
3xx Redirection 301 (Moved Permanently), 304 (Not Modified)
4xx Client Error 400 (Bad Request), 401 (Unauthorized), 404 (Not Found)
5xx Server Error 500 (Internal Server Error), 503 (Service Unavailable)

3. Comprehensive Status Codes Reference (30+ Codes)

Knowing these specific codes will help you debug your code much faster!

2xx Success (Everything is Good)

Code Status Meaning
200OKSuccess for GET/PUT requests.
201CreatedSuccess for POST (new resource created).
202AcceptedRequest accepted but processing is not complete.
204No ContentSuccess, but no data returned (common for DELETE).
206Partial ContentUsed for range requests (e.g., streaming video).

3xx Redirection (The resource moved)

Code Status Meaning
301Moved PermanentlyResource is now at a new URL forever.
302FoundTemporary redirect to a different URL.
304Not ModifiedCached version is still valid; no need to re-download.
307Temporary RedirectRedirection while maintaining the same HTTP method.
308Permanent RedirectPermanent move while maintaining the same HTTP method.

4xx Client Errors (Check your request)

Code Status Meaning
400Bad RequestServer cannot understand the request (Syntax error).
401UnauthorizedAuthentication is missing or invalid (Login failed).
403ForbiddenAuthenticated but not allowed to access this resource.
404Not FoundThe requested URL does not exist.
405Method Not AllowedUsing GET on a URL that only supports POST.
408Request TimeoutClient took too long to send the request.
409ConflictResource conflict (e.g., trying to use an existing email).
410GoneThe resource is deleted and no longer has a redirect.
413Payload Too LargeThe request data (or file) is too large for the server.
415Unsupported MediaSending the wrong format (e.g., XML instead of JSON).
422Unprocessable EntitySemantic errors (Valid JSON, but wrong field values).
429Too Many RequestsRate limiting (Requesting too fast).

5xx Server Errors (The server is struggling)

Code Status Meaning
500Internal Server ErrorThe server crashed or encountered an unknown error.
501Not ImplementedServer doesn't recognize the request method.
502Bad GatewayInvalid response from the upstream server.
503Service UnavailableDown for maintenance or overloaded.
504Gateway TimeoutThe upstream server took too long to respond.
505HTTP Version Not SupportedServer does not support the HTTP protocol version.

4. Runnable Example: Checking Status Codes

Use this code to see how JavaScript captures the status code from an API response.

<!DOCTYPE html> <html> <body> <button onclick="checkStatus()">Check API Status</button> <p id="result"></p> <script> async function checkStatus() { // Calling a post that exists (Status 200 expected) const response = await fetch('https://jsonplaceholder.typicode.com/posts/1'); // Accessing the status property const code = response.status; const ok = response.ok; // true if status is 200-299 document.getElementById("result").innerHTML = `Status Code: ${code}
Was successful? ${ok}`; } </script> </body> </html>

5. Detailed Methods Table

Method Usage Request Body?
GET Retrieving a list of users or data. No
POST Submitting a signup form or new data. Yes
PUT Replacing user profile data entirely. Yes
DELETE Removing a specific comment or file. No
Oppty Academy Tip: Idempotency means that making the same request multiple times will have the same result. For example, deleting a post once is the same as deleting it ten times—it stays deleted. This is why DELETE is idempotent, but POST is not (as it creates a new item every time).
