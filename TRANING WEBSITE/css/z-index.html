CSS z-index

The z-index property controls the vertical stacking order of elements that overlap. While HTML elements are normally flat, z-index manages the Z-axis (depth), determining which elements appear closer to the user and which remain in the background.

1. The z-index Value Matrix

Understanding how different values impact the stacking order is essential for managing complex UI layers like modals, tooltips, and dropdowns.

Value Type Behavior Stacking Logic
auto Default Value The element's stack order is the same as its parent.
Positive Number z-index: 100; Higher numbers move elements toward the front (closer to the user).
Negative Number z-index: -1; Lower numbers move elements behind the normal document flow.
initial / inherit Property Reset Sets the value to default or inherits from the parent.

2. Technical Architecture & Requirements

Before z-index can function, specific architectural conditions must be met within your CSS structure.

A. Position Requirement

Strict Rule: z-index only works on positioned elements. This includes any element with position set to relative, absolute, fixed, or sticky. It also works on flex/grid children.

B. Natural Stack Order

If two elements overlap and neither has a z-index, the element defined later in the HTML source code will naturally appear on top. This is known as the default stacking order.

C. Stacking Context (The "Folder" Logic)

A Stacking Context is an isolated hierarchy of layers. Once an element is inside a parent that has its own stacking context, its z-index is only relative to other elements in that same "folder."

Created By... Technical Condition
Positioning position (not static) + z-index (not auto).
Transparency Elements with opacity less than 1.
Transformation Elements with transform, filter, or perspective.

3. Interactive z-index Examples

Example 1: Basic Triple Layering

Three boxes overlapping. The Green box is defined first in HTML but stays on top due to the highest z-index.

<!DOCTYPE html> <html> <style> .box { position: absolute; width: 100px; height: 100px; padding: 10px; border: 2px solid #000; } .green { background: #04AA6D; z-index: 3; top: 20px; left: 20px; } .blue { background: #2196F3; z-index: 2; top: 50px; left: 50px; } .red { background: tomato; z-index: 1; top: 80px; left: 80px; } </style> <div class="box green">Z: 3</div> <div class="box blue">Z: 2</div> <div class="box red">Z: 1</div> </html>

Example 2: Negative z-index Depth

Using z-index: -1 to push a background shape behind the text flow.

<!DOCTYPE html> <html> <style> .bg-shape { position: absolute; width: 200px; height: 50px; background: gold; z-index: -1; transform: rotate(-5deg); } </style> <div class="bg-shape"></div> <h2>Highlighted Heading</h2> <p>The yellow bar is behind the text because of negative z-index.</p> </html>

Example 3: The Stacking Context Trap

Despite z-index: 9999, the pink box cannot overlap the grey box because its parent is locked at z-index: 1.

<!DOCTYPE html> <html> <style> .folder-a { position: relative; z-index: 1; background: #ddd; height: 100px; } .pink-item { position: absolute; z-index: 9999; background: pink; top: 20px; left: 120px; padding: 10px; } .folder-b { position: relative; z-index: 2; background: #999; height: 100px; margin-top: -30px; } </style> <div class="folder-a"> Parent A (Z: 1) <div class="pink-item">Child Z: 9999</div> </div> <div class="folder-b">Parent B (Z: 2)</div> </html>

Example 4: Floating Nav (Fixed UI)

A persistent header that uses a very high z-index to stay above all site content.

<!DOCTYPE html> <html> <style> .navbar { position: fixed; top: 0; width: 100%; background: #333; color: #fff; padding: 15px; z-index: 1000; } .page-content { position: relative; z-index: 1; margin-top: 60px; height: 1000px; background: #eee; padding: 20px; } </style> <div class="navbar">Fixed Header (Z-Index: 1000)</div> <div class="page-content">Scroll down to see content slide under the nav.</div> </html>

Example 5: Overlapping Gallery Cards

Using z-index to manage active vs. inactive states in a stacked card gallery.

<!DOCTYPE html> <html> <style> .card { position: absolute; width: 150px; height: 100px; border: 2px solid white; box-shadow: 0 4px 8px rgba(0,0,0,0.2); } .card1 { background: coral; z-index: 3; top: 0; left: 0; } .card2 { background: lightblue; z-index: 2; top: 20px; left: 40px; } .card3 { background: lightgreen; z-index: 1; top: 40px; left: 80px; } </style> <div style="position: relative; height: 150px;"> <div class="card card1">Card 1 (Z:3)</div> <div class="card card2">Card 2 (Z:2)</div> <div class="card card3">Card 3 (Z:1)</div> </div> </html>

4. Common Pitfalls & Solutions

Symptom Root Cause & Fix
z-index is being ignored. Check position. Elements must not be static.
Element hidden behind parent. Parent might have overflow: hidden or a conflicting context.
Higher z-index stays behind. Verify stacking contexts. The element might be stuck in a "lower folder."

Expert Tip: Avoid "z-index wars" by using a scale of 10. Set your base layers at 10, overlays at 100, and modals at 1000. This leaves room to add layers in between without rewriting your entire CSS file.
