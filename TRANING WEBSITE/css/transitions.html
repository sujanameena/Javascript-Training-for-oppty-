CSS Transitions

CSS transitions allow you to change property values smoothly (from one value to another), over a given duration. Instead of an instant change, transitions create a "tweening" effect that makes the UI feel organic and polished.

1. The Transition Property Matrix

To create a transition, you must specify two things: the property you want to change and the duration of the effect.

Property Description Values
transition-property The name of the CSS property to track. width, color, all
transition-duration How many seconds or milliseconds the transition takes. 0.3s, 500ms
transition-timing-function Specifies the speed curve (acceleration). ease, linear, cubic-bezier
transition-delay Wait time before the effect starts. 1s, 0.2s

2. Basic Syntax & Shorthand

While you can write properties individually, the shorthand is the industry standard for clean code:

/* transition: [property] [duration] [timing-function] [delay] */
.box { transition: background-color 0.5s ease-in-out 0.1s; }

3. 5 Interactive Transition Examples

Example 1: Color & Background Smoothness

A simple property change that makes buttons feel premium.

<!DOCTYPE html> <html> <head> <style> .color-box { width: 200px; padding: 20px; text-align: center; background-color: #667eea; color: white; transition: background-color 0.5s ease; cursor: pointer; border-radius: 8px; } .color-box:hover { background-color: #ff6b6b; } </style> </head> <body> <div class="color-box">Hover to change color</div> </body> </html>

Example 2: Transform & Scaling

Using transform is performant and creates great visual impact.

<!DOCTYPE html> <html> <head> <style> .transform-box { width: 100px; height: 100px; background: #4ecdc4; transition: transform 0.4s ease-in-out; display: flex; align-items: center; justify-content: center; color: white; } .transform-box:hover { transform: rotate(15deg) scale(1.2); } </style> </head> <body> <div class="transform-box">Grow & Rotate</div> </body> </html>

Example 3: Timing Functions Comparison

Compare how linear (constant speed) feels vs ease-in-out (smooth acceleration).

<!DOCTYPE html> <html> <head> <style> .bar { width: 100px; height: 30px; margin: 10px 0; color: white; padding-left: 10px; } .container:hover .bar { width: 300px; } .linear { background: red; transition: width 2s linear; } .ease-io { background: blue; transition: width 2s ease-in-out; } .cubic { background: green; transition: width 2s cubic-bezier(0.68, -0.55, 0.27, 1.55); } </style> </head> <body> <div class="container"> <div class="bar linear">Linear</div> <div class="bar ease-io">Ease-In-Out</div> <div class="bar cubic">Bouncy (Cubic)</div> <p>Hover here to start!</p> </div> </body> </html>

Example 4: Delayed Transitions

Using transition-delay to create a sequence of movements.

<!DOCTYPE html> <html> <head> <style> .delay-box { width: 100px; height: 100px; background: #ffd93d; transition: transform 0.5s ease; } .delay-box:hover { transform: translateX(200px); transition-delay: 0.5s; /* Wait half a second */ } </style> </head> <body> <div class="delay-box">Wait for it...</div> </body> </html>

Example 5: Multiple Property Transition

Transitioning width, height, and border-radius simultaneously.

<!DOCTYPE html> <html> <head> <style> .multi-box { width: 100px; height: 100px; background: #764ba2; transition: width 0.3s, height 0.3s, border-radius 0.3s, background 1s; } .multi-box:hover { width: 150px; height: 150px; border-radius: 50%; background: #667eea; } </style> </head> <body> <div class="multi-box"></div> </body> </html>

4. Pro-Tips & Best Practices

1. Performance: Always prefer transitioning transform and opacity. Transitioning width, height, or top/left causes the browser to recalculate the entire layout (reflow), which can be slow.

2. All vs. Specific: While transition: all is easy, it is better for performance to list specific properties like transition: color 0.2s, opacity 0.2s;.

3. Hidden to Visible: You cannot transition from display: none to display: block. Instead, transition opacity from 0 to 1 and visibility.
