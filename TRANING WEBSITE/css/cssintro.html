CSS Introduction & Syntax

CSS (Cascading Style Sheets) is the standard language used to style and layout web pages. It allows developers to control the visual appearance of HTML elements, including color, spacing, and positioning, while keeping the structure (HTML) separate from the presentation.

1. The CSS Rule-Set

A CSS rule consists of a selector and a declaration block. Understanding how these parts interact is crucial for writing efficient code.

  • Selector: Points to the HTML element you want to style (e.g., h1).
  • Declaration Block: Contains one or more declarations separated by semicolons and enclosed in curly braces { }.
  • Property: The specific style attribute you want to change (e.g., color, font-size).
  • Value: The setting assigned to the property (e.g., blue, 24px).
/* Selector { Property: Value; } */ h1 { color: blue; font-size: 24px; text-align: center; }

2. Three Ways to Apply CSS

CSS can be implemented in three different ways. Browsers resolve conflicts between these types using a hierarchy known as Cascading.

A. Inline CSS (Highest Priority)

Inline CSS involves applying styles directly to individual HTML elements using the style attribute. It overrides any external or internal styles.

When to Use:
  • For quick fixes and small changes that don't require a separate file.
  • When you need to override other styles for a specific element.
  • In HTML emails where external CSS support is limited.
<!DOCTYPE html> <html> <body> <p style="color:#009900; font-size:50px; font-style:italic; text-align:center;"> Inline CSS Example </p> </body> </html>

B. Internal or Embedded CSS (Medium Priority)

This is defined within the HTML document's <style> element, usually placed in the <head> section. It applies styles to elements within that specific file only.

When to Use:
  • When designing a single-page website.
  • If you need better control than inline CSS but don't have a multi-page project.
<!DOCTYPE html> <html> <head> <style> .main-div { text-align: center; } .heading-text { color: #009900; font-size: 50px; font-weight: bold; } .sub-text { font-style: italic; font-size: 20px; } </style> </head> <body> <div class="main-div"> <div class="heading-text">Internal CSS</div> <div class="sub-text">Implementation of Internal CSS</div> </div> </body> </html>

C. External CSS (Standard Method)

External CSS uses separate files with a .css extension containing only style properties. These files are linked to HTML documents using the <link> tag. It is the most efficient way to style multiple pages as it ensures consistent styling and faster loading due to caching.

When to Use:
  • For large-scale projects where multiple pages share a design.
  • When maintainability and scalability are priorities.
/* style.css content */ body { background-color: powderblue; } .main { text-align: center; } .GFG { color: #009900; font-size: 50px; font-weight: bold; } /* In your HTML file: */ <head> <link rel="stylesheet" href="style.css"> </head>

3. The Working of CSS

The browser follows a specific sequence to render your styles:

  1. Load HTML & Parse: The browser fetches the HTML and creates the DOM tree.
  2. Load & Parse CSS: External and internal styles are converted into the CSSOM (CSS Object Model).
  3. Build Render Tree: The DOM and CSSOM are combined into a tree of visible elements.
  4. Layout & Paint: The browser calculates exact positions (Layout) and draws the pixels (Paint).
