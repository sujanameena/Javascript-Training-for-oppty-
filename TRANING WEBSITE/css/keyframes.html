CSS Animations & @keyframes: Ultimate Master Class

CSS Animations allow you to create complex sequences of changes. Unlike transitions, which only move from state A to state B, animations use keyframes to control every stage of a movement cycle.

1. The Basic Syntax of @keyframes

The @keyframes rule is where you define the animation sequence. It acts as a "script" for the browser to follow.

@keyframes animationName {
  0% { transform: scale(1); opacity: 0; }
  50% { transform: scale(1.2); opacity: 0.5; }
  100% { transform: scale(1); opacity: 1; }
}
Syntax Part Explanation
@keyframesThe keyword that starts the animation definition.
animationNameThe custom name you give your animation (e.g., bounce).
Percentage (0%-100%)Points in time. 0% is the start, 100% is the completion.
from / toAliases for 0% (from) and 100% (to).

2. The Animation Property Matrix

Once the keyframe is defined, you apply it to an element using these sub-properties or the shorthand.

Property Purpose Common Values
animation-nameLinks the element to the @keyframes name.slideIn, bounce
animation-durationHow long the animation cycle lasts.0.5s, 2s, 5s
animation-timing-functionThe speed curve (acceleration).ease, linear, ease-in
animation-iteration-countHow many times to repeat.1, infinite, 5
animation-directionThe playback path.normal, reverse, alternate
animation-fill-modeStyle of element after finishing.forwards, backwards, both
animation-play-stateToggle between active/stopped.running, paused

3. The Transform Matrix

The transform property is most commonly used inside animations because it is highly performant.

Transform Type Example Syntax Result
Translatetransform: translate(50px, 30px);Moves element on X/Y axis.
Rotatetransform: rotate(45deg);Turns element clockwise.
Scaletransform: scale(1.5);Resizes element (larger/smaller).
Skewtransform: skew(20deg, 10deg);Distorts element angles.

4. Interactive Animation Examples

Example 1: Complex Bounce & Slide

Combining movement and opacity in a multi-step sequence. Explanation: Uses percentages (0%, 20%, 50%...) to define different heights during the bounce to mimic gravity.

<!DOCTYPE html> <html> <head> <style> @keyframes slideIn { 0% { transform: translateX(-100%); opacity: 0; } 100% { transform: translateX(0); opacity: 1; } } @keyframes bounce { 0%, 20%, 50%, 80%, 100% { transform: translateY(0); } 40% { transform: translateY(-30px); } 60% { transform: translateY(-15px); } } .box { width: 100px; height: 100px; background: #ff6b6b; color: white; display: flex; align-items: center; justify-content: center; border-radius: 8px; margin: 20px; } .slide-animation { animation: slideIn 1s ease forwards; } .bounce-animation { animation: bounce 2s infinite; } </style> </head> <body> <div class="box slide-animation">Slide In</div> <div class="box bounce-animation">Bounce</div> </body> </html>

Example 2: Pulse Effect (Keyframe Scale)

A soft scaling effect used for buttons or alerts. Explanation: The 50% keyframe increases the scale to 1.1, while 0% and 100% keep it at 1, creating a heartbeat effect.

<!DOCTYPE html> <html> <head> <style> @keyframes pulse { 0% { transform: scale(1); } 50% { transform: scale(1.1); } 100% { transform: scale(1); } } .pulse-box { width: 150px; height: 100px; background: #4ecdc4; color: white; display: flex; align-items: center; justify-content: center; border-radius: 5px; animation: pulse 1.5s infinite ease-in-out; } </style> </head> <body> <div class="pulse-box">Pulse Animation</div> </body> </html>

Example 3: Animation Shorthand (Spinning Loader)

Uses the 1-line shorthand for a rotating spinner. Explanation: linear timing ensures the speed never changes, making the rotation perfectly smooth.

<!DOCTYPE html> <html> <head> <style> @keyframes spin { 100% { transform: rotate(360deg); } } .shorthand-box { width: 80px; height: 80px; border: 5px solid #f3f3f3; border-top: 5px solid #ff6b6b; border-radius: 50%; animation: spin 1s linear 0s infinite normal none running; } </style> </head> <body> <div class="shorthand-box"></div> </body> </html>

Example 4: Color Morphing

Changing background colors across multiple steps. Explanation: Keyframes at 0, 25, 50, 75, and 100% allow the box to cycle through 4 distinct colors smoothly.

<!DOCTYPE html> <html> <head> <style> @keyframes morph { 0% { background: red; } 33% { background: blue; } 66% { background: green; } 100% { background: red; } } .morph-box { width: 100px; height: 100px; animation: morph 4s infinite; } </style> </head> <body> <div class="morph-box"></div> </body> </html>

Example 5: Shaking Error Alert

A high-speed animation used to indicate a wrong input. Explanation: Small translateX movements (5px) repeated quickly (0.1s) create the "vibration" effect.

<!DOCTYPE html> <html> <head> <style> @keyframes shake { 0%, 100% { transform: translateX(0); } 25% { transform: translateX(-5px); } 75% { transform: translateX(5px); } } .shake-box { padding: 20px; background: #eee; border: 2px solid red; display: inline-block; animation: shake 0.1s 5; } </style> </head> <body> <div class="shake-box">Invalid Login!</div> </body> </html>

Example 6: Floating Effect (Alternate)

Using alternate to move back and forth. Explanation: The direction: alternate property automatically reverses the animation after it hits 100%, making it return to 0% smoothly.

<!DOCTYPE html> <html> <head> <style> @keyframes float { from { transform: translateY(0px); } to { transform: translateY(-20px); } } .float-box { width: 100px; height: 100px; background: orange; border-radius: 50%; animation: float 2s infinite alternate ease-in-out; } </style> </head> <body> <div class="float-box"></div> </body> </html>

5. Pro-Tips & Summary

  • The "Forwards" rule: Use animation-fill-mode: forwards; so elements don't snap back to their original position after an animation ends.
  • Performance: Animating transform and opacity is much smoother than animating width or height.
  • Accessibility: Be careful with infinite loops. They can be distracting or cause nausea for some users.
