Difference between innerText and innerHTML

innerText and innerHTML are both properties of JavaScript. However, there are significant differences in how the text and HTML tags are handled when retrieving or setting content.

1. Getting Content: The Main Difference

The innerText property sets or returns the text content as plain text of the specified node and all its descendants, whereas the innerHTML property gets and sets the plain text or HTML contents in the elements.

<!DOCTYPE html> <html> <body style="text-align:center;"> <div id="test"> The following element contains some <code>code</code> and <i>some text</i>. </div> <br> <button onclick="innerTextFn()">Get innerText</button> <button onclick="innerHTMLFn()">Get innerHTML</button> <script> function innerTextFn() { let x = document.getElementById('test'); alert("innerText output: " + x.innerText); } function innerHTMLFn() { let x = document.getElementById('test'); alert("innerHTML output: " + x.innerHTML); } </script> </body> </html>

2. Setting Content with innerText

When assigning a string that contains HTML tags to the innerText property, it will automatically treat the tags as raw text, displaying them on the screen instead of rendering them.

<!DOCTYPE html> <html> <body> <h3>Using innerText to Update Content</h3> <p id="demo2">Original text.</p> <button onclick="changeText()">Change innerText</button> <script> function changeText() { // Tags will be printed visibly on the screen document.getElementById("demo2").innerText = "<b>This is bold text</b>"; } </script> </body> </html>

3. Setting Content with innerHTML

Unlike innerText, innerHTML lets you work with HTML rich text. It does not automatically encode and decode text, meaning any valid HTML passed to it will be parsed and styled by the browser.

<!DOCTYPE html> <html> <body> <h3>Using innerHTML to Update Content</h3> <p id="demo3">Original text.</p> <button onclick="changeHTML()">Change innerHTML</button> <script> function changeHTML() { // The browser will render the bold tag document.getElementById("demo3").innerHTML = "<b style='color:#ff7324'>This text is bold and colorful!</b>"; } </script> </body> </html>

4. Dynamic HTML Layout Generation

Because innerHTML parses markup, it is incredibly useful for generating complex layouts, tables, or lists dynamically based on JavaScript arrays.

<!DOCTYPE html> <html> <body> <h3>Generating a List</h3> <ul id="listContainer"></ul> <button onclick="generateList()">Load List</button> <script> function generateList() { const items = ["JavaScript", "HTML", "CSS"]; let markup = ""; for (let i = 0; i < items.length; i++) { markup += "<li>" + items[i] + "</li>"; } document.getElementById("listContainer").innerHTML = markup; } </script> </body> </html>

5. Security Considerations (Cross-Site Scripting)

Using innerHTML with external or user-provided data can lead to XSS (Cross-Site Scripting) attacks if malicious scripts are injected. Always use innerText when reflecting user input back onto the page.

<!DOCTYPE html> <html> <body> <h3>Safe Data Rendering</h3> <input type="text" id="userInput" value="<script>alert('Hacked!');</script>" size="40"> <br><br> <button onclick="safeDisplay()">Safe Display (innerText)</button> <p id="output"></p> <script> function safeDisplay() { const input = document.getElementById("userInput").value; // Using innerText ensures the script tag is just treated as text document.getElementById("output").innerText = "User wrote: " + input; } </script> </body> </html>
GeeksforGeeks Pro Tip: If you only need to read or change text content, using innerText is not only safer but often slightly faster than innerHTML because the browser doesn't have to parse the string for HTML tags.
