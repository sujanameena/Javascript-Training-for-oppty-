Semantic HTML Elements

Semantic elements clearly describe their meaning in a human- and machine-readable way, helping browsers and search engines understand page structure.

1. The <header> Element

The <header> element is used as a container for introductory content or a set of navigational links.

<header> <h1>Welcome to My Website</h1> <img src="logo.png" alt="Company Logo"> </header>
  • Usage: Typically contains headings (h1-h6), logos, or authorship info.
  • Rule: You can have several headers in one document, but they cannot be nested inside a footer or another header.

2. The <nav> Element

The <nav> element is specifically intended for major blocks of navigation links.

<nav> <a href="#home">Home</a> | <a href="#about">About</a> | <a href="#contact">Contact</a> </nav>
  • Purpose: Helps screen readers identify primary navigation areas.
  • Note: Not all links belong in <nav>; reserve it for main menus.

3. The <section> Element

The <section> element defines a thematic grouping of content, typically with a heading.

<section> <h2>Services We Offer</h2> <p>We provide web development and SEO consulting.</p> </section>
  • Usage: Used to split a page into logical chapters, news items, or contact info.

4. The <article> Element

The <article> element specifies independent, self-contained content.

<article> <h2>The Future of Web Design</h2> <p>Content of the blog post goes here...</p> </article>
  • Distinction: An article should make sense on its own and be distributable independently from the site.
  • Use Cases: Blog posts, forum posts, or product cards.

5. The <footer> Element

The <footer> element defines a footer for a document or a specific section.

<footer> <p>© 2026 My Website Inc.</p> <a href="contact.html">Contact Us</a> </footer>
  • Contents: Copyright data, sitemaps, and "back to top" links.
  • Flexibility: You can have multiple footers on a page (e.g., page-wide or within an article).
