HTML Document Structure

Every HTML page follows a specific nested structure that browsers use to render content correctly.

1. The Document Type Declaration (<!DOCTYPE html>)

This is the very first line of any HTML document.

<!DOCTYPE html> <html> ... </html>
  • Purpose: It is an instruction to the web browser about what version of HTML the page is written in.
  • Details: In modern web development, we use <!DOCTYPE html>, which specifically triggers "standard mode" for HTML5. It is not an HTML tag itself but a declaration.

2. The Root Element (<html>)

The <html> tag acts as the container for every other element on the page (except the DOCTYPE).

<html lang="en"> ... content goes here ... </html>
  • Purpose: It tells the browser where the HTML document begins and ends.
  • Attributes: The lang attribute (e.g., lang="en") helps search engines and screen readers identify the language of the page.

3. The Head Section (<head>)

The <head> tag contains metadata—information about the document that is not displayed directly to the user.

<head> <meta charset="UTF-8"> <title>My First Webpage</title> <link rel="stylesheet" href="style.css"> </head>
  • <title>: Sets the name shown in the browser tab.
  • <meta>: Defines character sets (UTF-8), keywords, and viewport settings.
  • <link>: Connects the HTML file to external resources like CSS.

4. The Body Section (<body>)

The <body> tag contains all the visible content of the webpage.

<body> <h1>Welcome!</h1> <p>This content is visible to users.</p> </body>
  • Purpose: Anything you want the user to see—text, images, videos, tables—must be placed here.
  • Structure: A well-structured body uses semantic tags to organize content into logical parts.

5. Semantic Structural Components (HTML5 Layout)

Modern HTML structure relies on semantic tags to divide the page into meaningful zones.

<header>Logo and Nav</header> <main> <article>Blog Post</article> </main> <footer>Copyright 2026</footer>
Tag Description
<header>Introductory content, logo, and navigation.
<nav>Specifically for navigation links.
<main>The unique, central content of the document.
<section>Groups related content together.
<footer>Bottom of page, copyright, and contact links.
