The addEventListener() Method

The addEventListener() method is the modern way to handle events in JavaScript. It allows you to attach multiple functions to a single event, works on any DOM object, and provides fine-grained control over how events "propagate" through the HTML tree.

1. Basic Syntax and Multiple Listeners

Unlike the old onclick property, addEventListener does not overwrite existing events. You can add two different functions to the same button click.

<!DOCTYPE html> <html> <body> <button id="btn">Click for Double Action</button> <script> const btn = document.getElementById("btn"); btn.addEventListener("click", () => alert("Action 1: Logging data...")); btn.addEventListener("click", () => alert("Action 2: Changing UI...")); </script> </body> </html>

2. Using the Event Object (e)

When an event occurs, the browser passes an Event Object to the handler. This object contains vital information like which element was clicked or the event type.

<!DOCTYPE html> <html> <body> <button id="infoBtn">Get Event Info</button> <p id="log"></p> <script> document.getElementById("infoBtn").addEventListener("click", (e) => { const msg = `Event Type: ${e.type} | Target: ${e.target.tagName}`; document.getElementById("log").innerText = msg; }); </script> </body> </html>

3. Event Bubbling vs. Capturing

Events in the DOM travel in two phases: Capturing (going down) and Bubbling (going up). By default, listeners trigger during Bubbling.

<!DOCTYPE html> <html> <body> <div id="parent" style="padding:50px; background:lightgrey;"> PARENT (Bubbles Last) <button id="child" style="display:block; margin:10px;">CHILD (Bubbles First)</button> </div> <script> document.getElementById("parent").addEventListener("click", () => alert("Parent clicked!")); document.getElementById("child").addEventListener("click", () => alert("Child clicked!")); </script> </body> </html>
