URL Params & Query Strings

When working with APIs, you often need to send extra information to the server, such as a search keyword, a page number, or a specific User ID. This is done using URL Parameters and Query Strings.

https://api.shop.com/products/101?color=blue&size=large

/products/101 ➔ URL Param (Specific Resource)
?color=blue&size=large ➔ Query String (Filters/Sorting)

1. The Difference: Params vs. Query Strings

Feature URL Parameters (Path) Query Strings
Placement Part of the URL path (/users/5) After the ? (?id=5)
Structure Values only Key-Value pairs (key=value)
Primary Use Identifying a specific item. Filtering, Sorting, Searching.

2. Deep Dive: Query String Syntax

A query string follows a very specific structure that browsers and servers understand:

  • The Question Mark (?): Marks the start of the query string.
  • Key-Value Pairs: Data is sent as key=value.
  • The Ampersand (&): Used to separate multiple pairs.
  • URL Encoding: Spaces and special characters are converted (e.g., a space becomes %20 or +).

3. The URLSearchParams API

The URLSearchParams interface defines utility methods to work with the query string of a URL. Instead of splitting strings manually, we use these methods:

  • get(name): Returns the first value associated with the given parameter.
  • set(name, value): Sets the value associated with a parameter.
  • append(name, value): Appends a new value for a parameter.
  • has(name): Returns a boolean indicating if a parameter exists.

4. Runnable Examples

Example 1: Reading Values from Current URL

This script detects parameters in the address bar. Note: This works best when the URL actually contains parameters.

<!DOCTYPE html> <html> <body> <p id="msg">No parameters detected.</p> <script> // Current URL: example.com?category=shoes&sort=price const params = new URLSearchParams(window.location.search); if(params.has('category')) { const cat = params.get('category'); document.getElementById("msg").innerHTML = `Category Found: <b>${cat}</b>`; } else { document.getElementById("msg").innerHTML = "Add <code>?category=electronics</code> to your URL to see this change!"; } </script> </body> </html>

Example 2: Dynamic API Path (URL Params)

Using Template Literals to inject a specific ID into the API path.

<!DOCTYPE html> <html> <body> <input type="number" id="postId" value="5"> <button onclick="getPost()">Fetch Post</button> <div id="result"></div> <script> async function getPost() { const id = document.getElementById("postId").value; const response = await fetch(`https://jsonplaceholder.typicode.com/posts/${id}`); const data = await response.json(); document.getElementById("result").innerHTML = `<p>Post Title: ${data.title}</p>`; } </script> </body> </html>

Example 3: Building a Query String from an Object

A clean way to convert a JavaScript object into a URL-ready string.

<script> const filterOptions = { search: "laptop", minPrice: 500, brand: "oppty" }; const encodedString = new URLSearchParams(filterOptions).toString(); document.write("Result: ?" + encodedString); // Output: ?search=laptop&minPrice=500&brand=oppty </script>

Example 4: Adding Multiple Values for One Key

Sometimes you need to send multiple values for the same filter (e.g., choosing multiple colors).

<script> let urlParams = new URLSearchParams(); urlParams.append("color", "red"); urlParams.append("color", "blue"); urlParams.append("color", "green"); document.write("Final Query: ?" + urlParams.toString()); // Output: ?color=red&color=blue&color=green </script>

Example 5: Filtering API Data with Query Strings

Using query strings to limit results from a real server.

<!DOCTYPE html> <html> <body> <button onclick="getLimitedData()">Fetch Only 3 Posts</button> <pre id="output"></pre> <script> async function getLimitedData() { // ?_limit=3 is a query string used by this API to filter data const response = await fetch('https://jsonplaceholder.typicode.com/posts?_limit=3'); const data = await response.json(); document.getElementById("output").innerText = JSON.stringify(data, null, 2); } </script> </body> </html>
Oppty Academy Tip: Use URL Parameters for "What" you want (e.g., User #5). Use Query Strings for "How" you want it displayed (e.g., Sorted by date, Page 2).
