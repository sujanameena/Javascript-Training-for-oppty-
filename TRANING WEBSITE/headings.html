HTML Headings

Headings are used to define the hierarchy and structure of your webpage content.

1. The Heading Hierarchy (H1 to H6)

HTML provides six levels of headings, which represent the importance and structure of the content.

<h1>Main Heading (H1)</h1> <h2>Sub-heading (H2)</h2> <h3>Section Heading (H3)</h3> <h4>Minor Heading (H4)</h4> <h5>Small Heading (H5)</h5> <h6>Least Important (H6)</h6>
  • <h1>: This is the most important heading. It is typically the main title of the page and has the largest font size (default: 32px or 2em).
  • <h2> to <h5>: These are sub-headings used to break down the content into smaller, readable sections.
  • <h6>: This is the least important heading, used for the smallest sub-sections.
  • GeeksforGeeks Tip: Always start with <h1> and work your way down. Never skip levels (e.g., don't go from h1 straight to h3).

2. Importance of Headings (SEO & Structure)

Headings are not just for making text big; they serve two critical functions:

<article> <h1>The Future of AI</h1> <p>Introduction text here...</p> <h2>Machine Learning Basics</h2> <p>Content about ML...</p> </article>
  • Search Engines (SEO): Search engines like Google use headings to index the structure and content of your web pages. A proper heading structure helps your site rank better.
  • User Experience: Headings allow users to "scan" a page quickly to find information. Without headings, a page would just be a wall of text.

3. Default Browser Styling

By default, browsers add specific styles to headings to distinguish them from regular text.

<h2>Default Styled Heading</h2> <p>Notice the bold font and the space (margin) above and below this paragraph.</p>
  • Boldness: All heading tags are rendered in a bold font weight.
  • Spacing: Browsers automatically add a margin (white space) before and after every heading to separate it from other elements.
  • Block Level: Headings are "block-level" elements; they always start on a new line and take up the full width available.

4. Changing Heading Size (CSS)

While each heading has a default size, GeeksforGeeks emphasizes that you should never use a heading tag just to make text large.

<!-- Use CSS for custom sizing --> <h1 style="font-size:60px;">Custom Large Heading</h1> <h2 style="font-size:12px;">Small heading with H2 semantics</h2>
  • Font Size: If you want a specific size, use the CSS font-size property rather than choosing a heading level based on its appearance.

5. Best Practices

To write clean, professional HTML, follow these rules from the tutorial:

<h1>Main Topic</h1> <hr> <h2>Section One</h2> <p>Details...</p>
  • One H1 Per Page: It is standard practice to use only one <h1> tag per webpage to represent the main topic.
  • Accessibility: Screen readers use headings to help visually impaired users navigate. Using headings in the correct order (h1 → h2 → h3) is essential.
  • The Horizontal Rule (<hr>): Often used alongside headings to create a thematic break or a horizontal line to separate sections visually.
