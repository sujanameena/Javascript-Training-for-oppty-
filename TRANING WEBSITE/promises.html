JavaScript Promises

A Promise in JavaScript is an object representing the eventual completion or failure of an asynchronous operation.

1. Promise States

A Promise is always in one of these three states:

  • Pending: The initial state before the operation finishes.
  • Fulfilled: The operation completed successfully (triggers .then()).
  • Rejected: The operation failed (triggers .catch()).
const myPromise = new Promise((resolve, reject) => { let success = true; if (success) { resolve("Operation Successful!"); } else { reject("Operation Failed!"); } }); myPromise .then((value) => { console.log(value); }) .catch((error) => { console.log(error); });

2. Using .then() and .catch()

The .then() method handles success, while .catch() handles any errors that occur during the process.

<script> let promise = new Promise(function(resolve, reject) { setTimeout(() => resolve("Data Loaded!"), 2000); }); promise.then( result => alert(result), error => alert(error) ); </script>

3. Chaining Promises

You can chain multiple .then() methods to perform a sequence of asynchronous tasks one after another.

<script> new Promise(function(resolve, reject) { setTimeout(() => resolve(1), 1000); }).then(function(result) { console.log(result); // 1 return result * 2; }).then(function(result) { console.log(result); // 2 return result * 2; }).then(function(result) { console.log(result); // 4 }); </script>
Oppty Academy Tip: Promises help you avoid "Callback Hell"—a situation where code becomes nested and unreadable. They make your code cleaner and easier to debug!
