HTML Form Validation

Client-side validation ensures that data submitted by the user matches the required format before it is sent to the server.

1. The required Attribute

This is the simplest form of validation. It ensures that an input field is not left empty when the user clicks the submit button.

<form> <label>Username:</label> <input type="text" name="username" required> <input type="submit"> </form>
  • How it works: The browser blocks submission and displays a warning if the field is empty.

2. Length Validation (minlength & maxlength)

These attributes control the number of characters a user can enter into a text-based input.

<label>Password (8-20 chars):</label> <input type="password" minlength="8" maxlength="20" required>
  • minlength: Specifies the minimum characters required.
  • maxlength: Sets a hard limit on the maximum characters allowed.

3. Type-Based Validation

By choosing a specific type, you trigger the browser's automatic internal validation rules for that data format.

<input type="email" placeholder="Enter Email"> <input type="url" placeholder="Enter Website URL">
  • Email: Checks for a valid structure (includes an @ and a domain).
  • URL: Ensures the entry starts with a protocol like http:// or https://.

4. Range Validation (min & max)

Used specifically for numeric or date inputs to ensure the value falls within a certain range.

<label>Age (18-99):</label> <input type="number" name="age" min="18" max="99">
  • min: Sets the lowest possible value.
  • max: Sets the highest possible value.

5. Pattern Validation (pattern)

The pattern attribute allows you to create custom validation rules using Regular Expressions (Regex).

<label>Phone (123-456-7890):</label> <input type="tel" pattern="[0-9]{3}-[0-9]{3}-[0-9]{4}" title="Format: 123-456-7890" required>
  • Pro Tip: Use the title attribute to provide a "tooltip" explanation if the user enters the format incorrectly.

Summary of Validation Attributes

Attribute Purpose Used For
requiredEnsures field is not emptyAny input
minlength / maxlengthValidates character countText, Password, Tel
min / maxValidates numeric/date rangeNumber, Date, Range
typeValidates formatEmail, URL, Date
patternValidates custom RegexText, Tel, Email
