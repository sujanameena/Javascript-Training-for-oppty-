HTML Accessibility (ARIA)

ARIA (Accessible Rich Internet Applications) is a set of attributes that make web content more accessible to people with disabilities, especially those using screen readers.

1. ARIA Roles (role="...")

Roles define what an element is or does. They are essential when using non-semantic elements like <div> to act as interactive components.

<div role="button" onclick="alert('Clicked!')" style="cursor:pointer;"> Click Me </div>
  • Purpose: Tells the screen reader the function of a widget.
  • Common Roles: role="button", role="navigation", role="search", and role="alert".

2. ARIA Labeling (aria-label & aria-labelledby)

These attributes provide a text label for elements that don't have visible text on the screen, such as icon buttons.

<!-- Label for an icon button --> <button aria-label="Close">X</button> <!-- Labeling using another element's ID --> <h2 id="header-id">Newsletter</h2> <div role="form" aria-labelledby="header-id">...</div>

3. ARIA Descriptions (aria-describedby)

This attribute links an element to more detailed information, like helper text or error messages.

<label>Password:</label> <input type="password" aria-describedby="pw-hint"> <p id="pw-hint">Password must be 8 characters long.</p>

4. ARIA States (aria-expanded & aria-hidden)

States are dynamic attributes that reflect the current condition of interactive elements.

<!-- Used for accordions or menus --> <button aria-expanded="false">Show Menu</button> <!-- Used to hide decorative icons from screen readers --> <span class="icon-star" aria-hidden="true"></span>

5. ARIA Live Regions (aria-live)

Live regions notify users of dynamic content updates that happen without a page reload.

<div aria-live="polite"> Item added to cart! </div>
  • polite: Waits until the screen reader finishes current speech.
  • assertive: Interrupts the screen reader immediately (for critical errors).
