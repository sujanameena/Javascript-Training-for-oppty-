Introduction to APIs & REST

An API (Application Programming Interface) is a software intermediary that allows two applications to talk to each other. Think of it as a waiter in a restaurant who takes your order (Request) to the kitchen (Server) and brings back your food (Response).

Client
(Mobile/Web App)
→ Request →
API Layer
(The Waiter)
→ Request →
Server
(Database/Logic)

Data flows back to the Client in the reverse direction as a Response.

1. Understanding API Architecture

Modern web development relies on the Client-Server Architecture. The client (frontend) and server (backend) are often hosted on totally different computers. The API is the "contract" that defines how they talk.

Feature API Role
Abstraction You don't need to know how the server's database works; you just ask the API for "Users".
Security APIs act as a gatekeeper, ensuring only authorized people get data.
Automation APIs allow software to trigger actions without human intervention.

2. Types of APIs

APIs are categorized based on who can access them and how they are used:

Type Description
Open APIs Public APIs, available for any developer to use.
Internal APIs Private APIs used only within a specific company.
Partner APIs Shared between business partners (e.g., Amazon and FedEx).

3. What makes an API "RESTful"?

REST (REpresentational State Transfer) is an architectural style. To be "RESTful," an API must follow these rules:

  • Statelessness: Every request is independent. The server doesn't remember past requests.
  • Resource-Based: Data (like users or books) is identified by unique URLs.
  • HTTP Methods: Uses standard verbs like GET, POST, PUT, and DELETE.

4. Deep Dive: API Endpoints

An Endpoint is the specific digital address where your API lives. Example structure:

// Base URL: https://api.opptybookstore.com/v1

GET /books (Read all books)
POST /books (Create a new book)
DELETE /books/101 (Delete book #101)

5. How an API Works (The 4-Step Process)

Step 1: The Request — The client sends a request (e.g., "GET /weather").
Step 2: Analysis — The API receives the request and validates it.
Step 3: Execution — The server processes the logic and fetches data.
Step 4: The Response — The API delivers the data (JSON) back to the client.

6. Real-World API Examples

Top 5 General APIs

  1. Weather: Sending data to your phone's home screen.
  2. Stock Market: Powering real-time trading apps.
  3. Maps: Helping Uber/Swiggy track locations.
  4. Log-in: "Sign in with Google" buttons.
  5. PayPal: Processing payments in online stores.

Top 5 REST APIs for Devs

  1. JSONPlaceholder: Free fake data for testing.
  2. NASA: Real satellite and space imagery data.
  3. TMDB: Movie posters, ratings, and cast info.
  4. Stripe: Handling credit card payments.
  5. GitHub: Managing code and user repositories.

7. Runnable API Examples (Try it Yourself)

These examples talk to a real server at jsonplaceholder.typicode.com.

Example 1: Fetching Data (GET)

<!DOCTYPE html> <html> <body> <h3>API Result:</h3> <p id="display">Loading...</p> <script> fetch('https://jsonplaceholder.typicode.com/posts/1') .then(response => response.json()) .then(data => { document.getElementById("display").innerHTML = "Title: " + data.title; }); </script> </body> </html>

Example 2: Fetching a List of Users

<!DOCTYPE html> <html> <body> <div id="user-info" style="padding:10px; border:1px solid #ccc;"> <h4 id="uname"></h4> <p id="uemail"></p> </div> <script> fetch('https://jsonplaceholder.typicode.com/users') .then(res => res.json()) .then(users => { document.getElementById("uname").innerText = "Name: " + users[0].name; document.getElementById("uemail").innerText = "Email: " + users[0].email; }); </script> </body> </html>

Example 3: Sending Data (POST)

<!DOCTYPE html> <html> <body> <button onclick="sendData()">Click to Create Post</button> <p id="status"></p> <script> function sendData() { fetch('https://jsonplaceholder.typicode.com/posts', { method: 'POST', body: JSON.stringify({ title: 'Hello', body: 'API Test', userId: 1 }), headers: { 'Content-type': 'application/json' } }) .then(res => res.json()) .then(json => { document.getElementById("status").innerHTML = "Created with ID: " + json.id; }); } </script> </body> </html>

Example 4: Error Handling (404)

<script> fetch('https://jsonplaceholder.typicode.com/posts/999') .then(response => { if (!response.ok) throw new Error("Post not found (404)"); return response.json(); }) .catch(error => { document.write("Caught Error: " + error.message); }); </script>

Example 5: Complex JSON Object (Weather Structure)

{ "city": "Mumbai", "coord": { "lon": 72.8, "lat": 19.07 }, "main": { "temp": 28.5, "humidity": 65 }, "weather": [{ "main": "Cloudy" }] }

8. API Protocols: REST vs SOAP

REST (Modern)

Lightweight, uses JSON, easy to scale. Used by 90% of modern web apps.

SOAP (Legacy)

Heavy, uses XML, highly secure. Preferred by banks and old corporate systems.

Oppty Academy Tip: Most REST APIs communicate using JSON. In JavaScript, you can use JSON.parse() to turn text into an object, and JSON.stringify() to turn an object into text for the API!
