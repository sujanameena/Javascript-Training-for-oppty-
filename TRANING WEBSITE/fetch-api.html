JavaScript Fetch API

The Fetch API is a modern, powerful interface for making network requests in JavaScript. It allows you to fetch resources asynchronously across the network using Promises, replacing the older XMLHttpRequest.

Syntax: fetch(url, options)
It returns a Promise that resolves to the Response object representing the response to your request.

1. HTTP Methods Comparison

When using the Fetch API, you must specify the "Method" to tell the server what action you want to perform.

Method Action Example Scenario
GET Retrieve data Fetching a user's profile picture or a list of posts.
POST Submit/Create data Registering a new user or sending a contact form.
PUT Update (Full) Replacing an entire existing record with new data.
PATCH Update (Partial) Changing only the 'email' field of a user profile.
DELETE Remove data Deleting a blog post or removing an item from a cart.

2. Basic GET Request

By default, fetch() performs a GET request. We use .json() to extract the JSON content from the response.

<!DOCTYPE html> <html> <body> <h2>Fetch GET Example</h2> <p id="output">Fetching data...</p> <script> fetch('https://jsonplaceholder.typicode.com/todos/1') .then(response => response.json()) .then(data => { document.getElementById("output").innerHTML = "Task: " + data.title + "<br>Completed: " + data.completed; }) .catch(error => console.log('Error:', error)); </script> </body> </html>

3. POST Request (Sending Data)

To send data, we provide an options object containing the method, headers, and the body (stringified JSON).

<!DOCTYPE html> <html> <body> <button onclick="createPost()">Send POST Request</button> <p id="post_res"></p> <script> function createPost() { fetch('https://jsonplaceholder.typicode.com/posts', { method: 'POST', body: JSON.stringify({ title: 'Oppty Academy', body: 'Learning Fetch API', userId: 1 }), headers: { 'Content-type': 'application/json; charset=UTF-8', }, }) .then(response => response.json()) .then(json => { document.getElementById("post_res").innerText = "New Resource ID: " + json.id; }); } </script> </body> </html>

4. PUT and DELETE Requests

PUT is used to update existing data, and DELETE is used to remove it. Note that most testing APIs (like JSONPlaceholder) "fake" the success.

<script> // Example of a DELETE Request fetch('https://jsonplaceholder.typicode.com/posts/1', { method: 'DELETE', }) .then(res => { if(res.ok) console.log("Post Deleted Successfully"); }); // Example of a PUT (Update) Request fetch('https://jsonplaceholder.typicode.com/posts/1', { method: 'PUT', body: JSON.stringify({ id: 1, title: 'Updated Title', body: 'Updated Content', userId: 1 }), headers: { 'Content-type': 'application/json' } }) .then(response => response.json()) .then(data => console.log("Updated Data:", data)); </script>

5. Async / Await with Fetch

Modern developers prefer async/await because it makes asynchronous code look cleaner and easier to read.

<!DOCTYPE html> <html> <body> <h3>Async/Await Fetch</h3> <div id="user_list"></div> <script> async function getUsers() { try { let response = await fetch('https://jsonplaceholder.typicode.com/users'); let users = await response.json(); let output = "<ul>"; users.slice(0, 3).forEach(user => { output += `<li>${user.name} - ${user.email}</li>`; }); output += "</ul>"; document.getElementById("user_list").innerHTML = output; } catch (error) { console.error("Fetch Error:", error); } } getUsers(); </script> </body> </html>
Oppty Tip: Always check response.ok. A fetch promise won't reject on HTTP errors like 404 or 500; it only rejects on network failures!
