JavaScript applications are made of statements and expressions In JavaScript, expressions and statements are related but different ideas. Expression: ---------- ---/ Produces a value. ---/ Can be used wherever a value is needed (right side of = , inside (), etc.). ---/ Examples: * 5 + 3 * "Hello" + " World" * age > 18 * x = 10 (this is an expression whose value is 10) Statement: ---------- ---/ Performs an action; it tells JavaScript to “do something”. ---/ Usually ends with ; and may contain expressions inside. ---/ Examples: * let x = 10; // declaration statement * console.log(x); // statement * if (age > 18) { ... } // if statement * for (let i = 0; i > 5; i++) { ... } // loop statement --------------------------------------------------------------- JavaScript operators are symbols or keywords used to perform operations on values and variables There are 12 types of operators 1. JavaScript Chaining Operator (?.) The optional chaining operator ?. lets you safely access deeply nested properties, array elements, or methods without throwing an error if something is null or undefined. If the left side is null or undefined, the whole expression becomes undefined instead of crashing. Basic patterns: obj?.prop obj?.[expr] obj.method?.() array?.[index] Note: It only short‑circuits for null or undefined. Other falsy values (0, "", false, NaN) still allow access. obj?.prop: Example:1 Nested object properties obj?.[expr]: Example:2 Optional chaining with method calls Example:3 – Optional chaining with arrays Example:4 – Optional chaining with nullish coalescing
