CSS Units Explained ******************** 1. px — Pixels (Fixed) Not responsive — stays fixed even if screen size changes. Exactly 1 pixel on screen. Fixed. Never changes. 1px = 1px always 20px = 20px always 2. % — Percentage (Relative to Parent) Depends on the parent element's width. Parent = 1000px → 10% = 100px Parent = 500px → 10% = 50px Parent = 200px → 10% = 20px 3.em — Relative to Current Element's Font Size Default browser font-size = 16px, so 1em = 16px by default. font-size: 16px → 1em = 16px, 2em = 32px, 0.5em = 8px font-size: 20px → 1em = 20px, 2em = 40px, 0.5em = 10px font-size: 10px → 1em = 10px, 2em = 20px, 0.5em = 5px 4.rem — Relative to Root Font Size Root default = 16px (unless you change it). html { font-size: 16px; } ← default 1rem = 16px 2rem = 32px 0.5rem = 8px 1.5rem = 24px 5. vw — Viewport Width 1vw = 1% of the browser window's width. Browser width = 1200px → 1vw = 12px, 5vw = 60px, 100vw = 1200px Browser width = 800px → 1vw = 8px, 5vw = 40px, 100vw = 800px 6. vh — Viewport Height 1vh = 1% of the browser window's height. Browser height = 900px → 1vh = 9px, 50vh = 450px, 100vh = 900px Browser height = 600px → 1vh = 6px, 50vh = 300px, 100vh = 600px Real-World Rule of Thumb: ************************* px → borders, box-shadows % → widths, responsive containers rem → font sizes, spacing (most recommended) em → component-local spacing (padding relative to its own text) vw/vh → hero sections, full-screen layouts Because of this only the previous one doesn’t worked

Example CSS Min and Max Dimensions

CSS Min/Max Dimensions

1. Width (Min & Max)

max-width: Prevents an element from becoming wider than a specific value. Great for mobile responsiveness.

width: 100%; max-width: 500px;
I will fill the screen, but I will never get wider than 500px.

min-width: Ensures an element is at least a certain width, even if the content or parent is smaller.

width: 10%; min-width: 200px;
Even though I want to be 10% wide, I will stop shrinking once I hit 200px.

2. Height (Min & Max)

min-height: Ensures an element is at least a certain height. It can still grow taller if there is more content.

min-height: 100px;
I am at least 100px tall. If you add more text, I will grow to fit it.

max-height: Limits how tall an element can grow. If content exceeds this, it will spill out (unless overflow is handled).

max-height: 150px;

Why use them? -------------- min-width / min-height: Prevent elements from becoming too small, even if the parent or content shrinks. This keeps content readable and layouts usable. max-width / max-height: Prevent elements from growing too large, which helps maintain design consistency and prevents overflow issues. What happens if you don’t use them? ------------------------------------ Elements can shrink to 0 (or as small as their content allows) or grow as large as their parent/container allows. On small screens, elements might become too narrow to read or use. On large screens, elements might stretch too wide or tall, breaking the design or making content hard to read.
