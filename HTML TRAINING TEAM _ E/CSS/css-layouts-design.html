Basic display values: block: --------- The display: block property in CSS makes an element a block-level element. Block-level elements: * Start on a new line. * Stretch to fill the full width of their parent container by default. * Allow you to set width, height, margin, and padding. * Stack vertically, one after another. * Common block-level elements include: div, p, h1–h6, ul, ol, li, and form. inline --------- The CSS property display: inline makes an element an inline element. Here’s what that means: * The element does not start on a new line; it flows alongside other inline elements, like text. * Only takes up as much width as its content. * You cannot set width or height on inline elements (those properties are ignored). * Padding, margin, and border only push content horizontally, not vertically. * Common inline elements: span, a, strong, em, img. Example: This is inline So is this inline-block ------------- The CSS property display: inline-block makes an element behave like an inline element (it flows alongside text and other inline elements), but it also allows you to set width and height, like a block element. Key points: The element sits inline with other elements (does not start on a new line). You can set width, height, margin, and padding. Useful for creating buttons, navigation links, or layouts where you want block-like styling but inline flow. Example: Inline-block element example Understanding the Display Property 1. *********************** CSS z-index Property *********************** Definition: The z-index property in CSS controls the vertical stacking order of elements that overlap. It only works on elements that have a position value other than static (i.e., position: relative, absolute, fixed, or sticky). Syntax: .element { position: relative; /* or absolute, fixed, sticky */ z-index: 2; /* any integer (positive, negative, or zero) */ } How it works: * Elements with a higher z-index value appear in front of those with a lower value. * z-index only works on positioned elements (not position: static). * z-index can be negative, zero, or positive. * Stacking context: A new stacking context is formed by elements with a position and a z-index value, opacity less than 1, certain CSS properties, or the root element. Example 1: Basic Overlap

Negative z-index can place elements behind others.
