***************************** CSS justify-content Property ***************************** Syntax justify-content: flex-start | flex-end | center | space-between | space-around | space-evenly | initial | inherit; justify-content Values: flex-start: Items are packed toward the start of the main axis (default). flex-end: Items are packed toward the end of the main axis. center: Items are centered along the main axis. space-between: Items are evenly distributed; first item at the start, last at the end, space between. space-around: Items are evenly distributed with equal space around them (half-size space on ends). space-evenly: Items are distributed so the space between any two items, and the space to the edges, is equal. initial: Sets the property to its default value (flex-start). inherit: Inherits the value from its parent.

***************************** CSS align-items Property ***************************** align-items: normal | stretch | center | flex-start | flex-end | baseline | first baseline | last baseline | start | end | self-start | self-end | safe center | unsafe center | initial | inherit; The CSS property align-items is used in flex and grid layouts to align items along the cross axis (vertical by default in flexbox). Here’s what each value does: normal: Default. Behaves as stretch for flexbox. stretch: Items stretch to fill the container (default for flexbox). center: Items are centered along the cross axis. flex-start: Items are aligned to the start of the cross axis. flex-end: Items are aligned to the end of the cross axis. baseline: Items are aligned so their baselines align. first baseline: Aligns items to the first baseline of the container. last baseline: Aligns items to the last baseline of the container. start: Aligns items to the start edge of the container (writing mode aware). end: Aligns items to the end edge of the container. self-start: Aligns items to their own start edge. self-end: Aligns items to their own end edge. safe center: Like center, but prevents overflow if possible. unsafe center: Like center, but allows overflow. initial: Sets to default value. inherit: Inherits from parent. ***************************** CSS Animations ***************************** @keyframes Rule ------------------ 1. Percentage keyframes: You can use percentages (from 0% to 100%) to define the animation steps. @keyframes example { 0% { background: red; } 50% { background: yellow; } 100% { background: green; } } 2. from and to keywords: from is the same as 0%, and to is the same as 100%. Example: @keyframes fadeIn { from { opacity: 0; } to { opacity: 1; } } 3. Multiple keyframes: You can define as many intermediate steps as you want using percentages. @keyframes bounce { 0% { transform: translateY(0); } 30% { transform: translateY(-50px); } 50% { transform: translateY(0); } 100% { transform: translateY(0); } }

animation-iteration-count property ----------------------------------- The CSS animation-iteration-count property specifies how many times an animation should repeat. Common values: A number (e.g., 1, 2, 3, etc.): The animation will play that many times. infinite: The animation will repeat forever. How it works: The blue box will bounce up and down 3 times, then stop. Set animation-iteration-count: infinite; to make it repeat forever. animation-direction property ----------------------------------- The CSS animation-direction property specifies whether an animation should play forward, backward, or alternate directions on each cycle. Values: normal (default): Animation plays forward each cycle. reverse: Animation plays backward each cycle. alternate: Animation reverses direction every cycle (forward, then backward, etc.). alternate-reverse: Animation reverses direction every cycle, starting backward. Syntax: animation-direction: normal | reverse | alternate | alternate-reverse; How it works: "normal": Moves right, jumps back, repeats. "reverse": Moves left, jumps back, repeats. "alternate": Moves right, then left, smoothly. "alternate-reverse": Moves left, then right, smoothly. Example:

animation-fill-mode property ----------------------------------- The CSS animation-fill-mode property specifies how a CSS animation applies styles to its target before and after it is executing. Values: none (default): The element does not retain any animation styles before or after the animation. forwards: The element retains the styles set by the last keyframe after the animation ends. backwards: The element applies the styles of the first keyframe before the animation starts (during delay). both: The element follows the rules for both forwards and backwards. How it works: With animation-fill-mode: forwards;, the box stays at 200px wide and orange after the animation completes, instead of snapping back to its original state. Syntax: animation-fill-mode: none | forwards | backwards | both; Example:
forwards

***************************** CSS Animations ***************************** The CSS grid-template-columns property defines the number, size, and behavior of columns in a CSS Grid container. Syntax: grid-template-columns: ...; Each value creates a column. You can use fixed units (px, em), percentages, fractions (fr), or keywords like auto, min-content, max-content, and repeat(). 1. Fixed Width Columns grid-template-columns: 100px 200px 150px; Creates 3 columns: 100px, 200px, and 150px wide. 2. Fractional Units (fr) grid-template-columns: 1fr 2fr 1fr; 1fr: 2fr: 1fr means the middle column is twice as wide as the others, and all columns share available space. 3. Percentage Widths grid-template-columns: 50% 50%; Two columns, each taking half the container’s width. 4. auto, min-content, max-content grid-template-columns: auto 200px min-content max-content; auto: Size based on content or available space. min-content: Smallest possible size without overflow. max-content: Widest possible size without wrapping. 5. repeat() Function grid-template-columns: repeat(4, 1fr); Creates 4 equal columns, each taking 1 fraction of available space. 6. mix of units grid-template-columns: 100px 2fr 20% auto; Mixes fixed, fractional, percentage, and auto sizing. 7. minmax() Function grid-template-columns: minmax(150px, 1fr) 2fr; First column is at least 150px, but can grow up to 1fr.
