You create (declare) variables with let, const, or var: * let → changeable value (block scoped). * const → value that should not be reassigned (block scoped). * var → older way, function/global scoped (avoid in modern code). Variable names: * Must start with a letter, _, or $ (not a number). * Are case-sensitive (age and Age are different). --------------------------------------------------------------------------------------------------------- Data Types: --------------------------------------------------------------------------------------------------------- Primitive Data Types: Includes Number, String, Boolean, Null, Undefined, BigInt, and Symbol. --------------------- Non-Primitive Data Types: Includes Object, Array, and Function used to store complex data. Primitive Datatypes: 1. Number: Represents numeric values (integers and decimals). 2. String: Represents text enclosed in single or double quotes. 3. Boolean: Represents a logical value (true or false). 4. Undefined: A variable that has been declared but not assigned a value. 5. Null: Represents an intentional absence of any value. 6. Symbol: Represents unique and immutable values, often used as object keys. 7. BigInt: Represents integers larger than Number.MAX_SAFE_INTEGER. Example: 1.Numbers Example: 2. String: 3. BigInt Example: BigInt is a JavaScript primitive data type used to store very large integers that are bigger than Number.MAX_SAFE_INTEGER. It keeps the exact value without losing precision Example: Use case (where we use BigInt) BigInt is useful when you work with very large whole numbers, for example: Big order IDs or user IDs Large counts (views, likes, money in smallest units, etc.) Any case where you must not lose even 1 unit due to rounding/precision issues.
