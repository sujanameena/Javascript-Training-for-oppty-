Explanation: 1 ^ and $ anchor the start/end of the string. [^\s@]+ before @: one or more characters that are not whitespace or “@” (local part). @: requires the “@” symbol. [^\s@]+ after @: one or more non-space/non-“@” chars (domain). .: requires a dot. [^\s@]{2,}: at least two non-space/non-“@” chars (TLD; can include additional dots like “co.uk”). Explanation:2 I’ll break /^[^\s@]+@[^\s@]+\.[^\s@]+$/ into pieces. /.../ In JavaScript, regex literals are written between / and /. The real pattern is everything inside. ^ Anchor: “start of the string”. Means matching must begin at the very first character (no characters allowed before). [^\s@]+ (first part, before the @) [...] = character class: “one character from this set”. ^ (inside the brackets) = negation: “NOT these characters”. \s = any whitespace (space, tab, newline, etc.). @ = the at-sign. So [^\s@] = “any character that is not whitespace and not @”. + = “one or more of the previous token”. Together, [^\s@]+ = “one or more non-space, non-@ characters”. This is the “username/local part” before the @. @ (the middle) Literal @ symbol. Must be present once here. [^\s@]+ (second part, after the @) Same as before: “one or more characters that are not whitespace and not @”. This is the domain name before the dot, e.g. gmail in gmail.com. \. . normally means “any character”. \. escapes it, so it becomes a literal dot character .. [^\s@]+ (third part, after the dot) Again: “one or more non-whitespace, non-@ characters”. This is the “top-level domain” part, like com, org, in. $ Anchor: “end of the string”. Ensures nothing is allowed after that last part. Putting it all together: Must start at the beginning of the string. Some non-space, non-@ characters. Then exactly one @. Then some non-space, non-@ characters. Then a dot .. Then some non-space, non-@ characters. And then the string must end. So it matches basic email-like strings such as user123@gmail.com, but it’s still a simple email check (not a perfect one). Example on regex: Regex Form Validation
